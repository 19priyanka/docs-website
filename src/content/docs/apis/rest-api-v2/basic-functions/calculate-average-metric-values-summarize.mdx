By default, New Relic REST API calls return a series of metric data values based on [time range restrictions](/docs/apm/apis/requirements/extracting-metric-data#time). To obtain the average of these values, include `&summarize=true` in your API call. For example:

```
curl -X GET "https://api.newrelic.com/v2/applications/<var>$APPID</var>/metrics/data.json" \
     -H "
```

If you have an [EU region account](/docs/using-new-relic/welcome-new-relic/getting-started/introduction-eu-region-data-center), the above URL begins with `https://api.eu.newrelic.com`.

<Callout variant="tip">
  To prevent summarizing data, omit `summarize` in your API call. You do not need to specify `&summarize=false`.
</Callout>
