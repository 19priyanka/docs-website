OpenTelemetry is a flexible toolkit that you can implement in a variety of ways. We recommend a basic four-step approach for setting up OpenTelemetry with New Relic. Here's an overview of the process, followed by details for each step.

1. [Prerequisites](#prereqs)
2. [Instrument your service with OpenTelemetry](#instrument)
3. [Send your telemetry data to New Relic](#send)
4. [View your data in the New Relic UI](#view-data)

## Step 1. Prerequisites [#prereqs]

First things first:

* If we don’t already know you, sign up for a free [New Relic account](/docs/accounts/accounts-billing/account-setup/create-your-new-relic-account).
* Copy your [Account License Key](https://one.newrelic.com/launcher/api-keys-ui.launcher).

## Step 2. Instrument your service with OpenTelemetry [#instrument]

To get started, you instrument your service with OpenTelemetry. OpenTelemetry has language-specific products and SDKs to help you. Many languages offer out-the-box instrumentation for common libraries and frameworks. Each language also provides an API for further instrumenting your service manually.

Go to the repository for your language and follow the instructions to instrument your service. When you're done, return here to complete the next step of [sending your telemetry data to New Relic](#send).

## Step 3. Send your telemetry data to New Relic [#send]

The [OpenTelemetry Protocol](https://github.com/open-telemetry/opentelemetry-specification/blob/main/specification/protocol/otlp.md), or OTLP for short, is a general purpose telemetry data delivery protocol designed for the OpenTelemetry project, which describes how to encode and transmit telemetry data. This makes it a natural choice as a transport for OpenTelemetry data.

New Relic supports OTLP ingest (currently in pre-release). This allows you to export telemetry data directly to New Relic using an OTLP exporter.

### Send data directly to New Relic [#direct]

You can configure your service to send data directly to New Relic:

![Diagram showing OpenTelemetry using New Relic's OTLP endpoint.](./images/native_otlp.png "native_otlp.png")

1. Configure the OTLP exporter to add a header ( ```api-key``` ) whose value is your [Account License Key](https://one.newrelic.com/launcher/api-keys-ui.launcher).
2. Configure the endpoint for the exporter to point to New Relic. You will receive the endpoint URL when you sign up for the [pre-release program](https://docs.google.com/forms/d/e/1FAIpQLSdIJVEAYaP7TXe9LmQA64yIObGvt-nOiz5kXYsjxLBbvut_1A/viewform).

### Send data through an OpenTelemetry Collector (optional) [#collector]

The [OpenTelemetry Collector](https://opentelemetry.io/docs/collector/) is a configurable and extensible software component to receive, process, and export telemetry data. When you set up a collector, it can operate in multiple capacities: As a gateway, the collector receives data from a variety of sources and applies standard processing before exporting to some backend. As an agent, the collector is deployed on each host in an environment and can collect telemetry data about the host and processes running on it.

When you take this approach, you configure your services to send data via OTLP to the collector. Then, just like the language SDKs, you can configure your OpenTelemetry Collector with an [OTLP exporter](https://github.com/open-telemetry/opentelemetry-collector/tree/main/exporter/otlpexporter) to send data to New Relic.

Here's a diagram showing how a service sends data through an OpenTelemtry collector:

![Diagram showing OpenTelemetry using the OpenTelemetry Collector and New Relic's OTLP endpoint.](./images/native_otlp_with_collector.png "native_otlp_with_collector.png")

Here's a Docker example of how to set up and run an OpenTelemetry Collector with the collector YAML:

1. Save the following as `otel-config.yaml`:

```
receivers:
  otlp:
    protocols:
      grpc:
      http:

processors:
  batch:

exporters:
  otlp:
    endpoint: ${OTEL_EXPORTER_OTLP_ENDPOINT}
    headers:
      api-key: ${NEW_RELIC_LICENSE_KEY}

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
    metrics:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
    logs:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
```
2. Run the OpenTelemetry collector, making sure you replace <var>OTLP_ENDPOINT_HERE</var> with the endpoint you received when signing up for the [pre-release](https://docs.google.com/forms/d/e/1FAIpQLSdIJVEAYaP7TXe9LmQA64yIObGvt-nOiz5kXYsjxLBbvut_1A/viewform) and replace <var>YOUR_KEY_HERE</var> with your [Account License Key](https://one.newrelic.com/launcher/api-keys-ui.launcher):

```
export OTEL_EXPORTER_OTLP_ENDPOINT=<var>OTLP_ENDPOINT_HERE</var>
export NEW_RELIC_LICENSE_KEY=<var>YOUR_KEY_HERE</var>

docker run --rm \
  -e OTEL_EXPORTER_OTLP_ENDPOINT \
  -e NEW_RELIC_LICENSE_KEY \
  -p 4317:4317 \
  -v "${PWD}/otel-config.yaml":/otel-config.yaml \
  --config otel-config.yaml \
  --name otelcol \
  otel/opentelemetry-collector
```
