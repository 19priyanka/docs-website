### New

* The following integrations are now stable (v1.0):

  * [Azure Event Hub](https://docs.newrelic.com/docs/integrations/microsoft-azure-integrations/azure-integrations-list/azure-event-hub-monitoring-integration)
  * [Azure Front Door](https://docs.newrelic.com/docs/integrations/microsoft-azure-integrations/azure-integrations-list/azure-front-door-monitoring-integration)
  * [Azure Logic Apps](https://docs.newrelic.com/docs/integrations/microsoft-azure-integrations/azure-integrations-list/azure-logic-apps-monitoring-integration)
  * [Azure Container](https://docs.newrelic.com/docs/integrations/microsoft-azure-integrations/azure-integrations-list/azure-containers-monitoring-integration)

  [](https://docs.newrelic.com/docs/integrations/microsoft-azure-integrations/azure-integrations-list/azure-containers-monitoring-integration)
* New metrics will be gathered for all Amazon ElastiCache clusters and nodes:
  * `cpuCreditUsage`
  * `cpuCreditBalance`
  * `databaseMemoryUsagePercentage`
* New metrics will be gathered for Amazon ElastiCache Redis nodes:
  * `cacheHitRate`
  * `db0AverageTtl`
  * `masterLinkHealthStatus`
  * `memoryFragmentationRatio`
  * `evalBasedCmds`
  * `evalBasedCmdsLatency`
  * `geoSpatialBasedCmds`
  * `geoSpatialBasedCmdsLatency`
  * `getTypeCmdsLatency`
  * `hashBasedCmdsLatency`
  * `hyperLogLogBasedCmdsLatency`
  * `keyBasedCmdsLatency`
  * `listBasedCmdsLatency`
  * `pubSubBasedCmds`
  * `pubSubBasedCmdsLatency`
  * `setBasedCmdsLatency`
  * `setTypeCmdsLatency`
  * `sortedBasedCmdsLatency`
  * `stringBasedCmdsLatency`
  * `streamBasedCmdsLatency`
