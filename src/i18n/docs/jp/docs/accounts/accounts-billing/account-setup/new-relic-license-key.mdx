New Relic ライセンスキーは、40文字の16進数の文字列で構成され、New Relicアカウントに[サインアップ](/docs/subscriptions/creating-your-new-relic-account)する時に作成されます。これはAPM とインフラストラクチャ データを送信するアカウントを決定するために使用されます（モバイルとブラウザは[他のキー](#other-keys)を使用します）。一部のNew Relic 機能では、ライセンスキーを入力する必要があります。

<Callout variant="important">
  貴社データへの不正アクセスを阻止するため、[ライセンスキーは安全に取扱う必要があります](#license-key-security)。
</Callout>

## ライセンスキーの表示と管理 [#finding]

New Relicアカウントのライセンスキーは、以下の手順で見ることができます。

1. [アカウント ドロップダウン](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown)から**Account settings（アカウントの設定）**を選択します。
2. ライセンスキーは、このページの右側に表示されます。

[追加のライセンスキーを作成し、NerdGraph API でそれらを管理することができます](/docs/apis/nerdgraph/examples/use-nerdgraph-manage-license-keys-personal-api-keys)。

<Callout variant="important">
  2019年12月から、ライセンスキーは、`NRAL`で終わる40文字の文字列となっています。
</Callout>

## ライセンスキーを安全に管理する [#license-key-security]

ライセンスキーは、アカウントとNew Relic に報告するデータを一意に識別します。このため、**パスワード同様に取り扱うことが非常に重要です**。

ライセンスキーが悪人の手に渡った場合、攻撃者は捏造したデータを貴社アカウントに送信することで、誤認アラートを引き起こすかデータを汚染させ、貴社が実際の問題を検出できないようにします。

ライセンスキーをセキュアに保護するためのアイデア：

* ライセンスキーは、設定ファイルにハードコードするのではなく、環境変数に保管してください。
* [Git Secrets](https://github.com/awslabs/git-secrets)または[truffleHog](https://github.com/dxa4481/truffleHog)などのツールを利用して、定期的にライセンスキーをスキャンしてください。
* [Vault](https://www.vaultproject.io/)または[CyberArk](https://www.cyberark.com/)などの秘密共有ツールを利用してください。

## Heroku ユーザー [#find-heroku]

Herokuアプリケーションをお持ちの場合、New Relic ライセンスキーを表示できます。

1. [Herokuダッシュボード](https://dashboard.heroku.com)にログインします。
2. アプリケーションを選択し、**Add-ons > New Relic APM**を選択します。
3. New Relic UIの[アカウントドロップダウン](/docs/accounts-partnerships/education/getting-started-new-relic/glossary#account-dropdown)から**Account settings**を選択します。
4. **Account Information**セクションから、ライセンスキーをコピーします。
