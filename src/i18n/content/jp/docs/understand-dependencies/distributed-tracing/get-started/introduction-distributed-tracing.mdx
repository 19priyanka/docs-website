ディストリビューティッド（分散）トレーシングは、APMトランザクショントレーシングを拡張します。トランザクショントレーシングは、1つのアプリケーションのアクティビティを分析するのに効果的です。ディストリビューティッド（分散）トレーシングを使用すると、リクエストが分散システムを通過する際の移動全体を確認できます。

<Callout variant="tip">
  ディストリビューティッド（分散）トレーシングとAPM、およびその他の[オブザーバビリティプラットフォーム](https://one.newrelic.com)を使用するには、New Relicファミリーに参加してください。[サインアップ](https://newrelic.com/signup)して、わずか数秒で無料アカウントを作成できます。次に、毎月最大で100GBのデータを無料で取り込みます。期間無制限です。
</Callout>

## アプリとサービスにおけるトレースリクエスト [#features]

APMトランザクショントレースは単一アプリケーションでの呼び出しの分析に役立つ一方、ディストリビューティッド（分散）トレーシングは、リクエストが複数のアプリケーションとサービスにわたってたどるパスを最初から最後まで示すことで表示範囲を広げます。この広い視野で、パフォーマンスのボトルネックとエラーをすばやくトラブルシューティングできます。

リクエストがサービス間を移動する際、各セグメントはスパンとして記録されます。スパンは、サービスやサービスのリソースで費やされた時間を表します。リクエストのすべてのスパンは、リクエスト全体を把握できるよう、単一のディストリビューティッド（分散）トレーシングに組み合わされます。
