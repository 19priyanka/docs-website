当社では、御社の分散システムを監視・分析するためのディストリビューティッド（分散）トレーシングを提供しています。

## クイックスタート [#enable]

直ちに開始する：

* [確認:](/docs/understand-dependencies/distributed-tracing/get-started/how-new-relic-distributed-tracing-works#sampling) さまざまな種類のディストリビューティッド（分散）トレーシングを確認してください。
* [有効化：](/docs/understand-dependencies/distributed-tracing/enable-configure/overview-enable-distributed-tracing) ディストリビューティッド（分散）トレーシングの設定方法をご覧ください。
* [使用：](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data) ディストリビューティッド（分散）トレーシングを有効にした後、UIでトレースを表示します。

## ディストリビューティッド（分散）トレーシングが重要な理由 [#features]

ソフトウェアのリクエストをエンドツーエンドで把握できると、トラブルシューティングをより簡単かつ迅速に行えます。ディストリビューティッド（分散）トレーシングでは、リクエストが分散システムを経由するパスを表示できるので、それが可能です。

リクエストがサービス間を移動する際、各セグメントはスパンとして記録されます。スパンは、サービスやサービスのリソースで費やされた時間を表します。リクエストのすべてのスパンは、リクエスト全体を把握できるよう、単一のディストリビューティッド（分散）トレーシングに組み合わされます。

トレースをNew Relicで把握すると、当社の強力なツールを使用して分析できます：

* [結果の絞り込み:](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data) 多くのデータポイントを使用してチャートを絞り込み、さまざまな方法でトレースデータを分析できます。
* [異常なスパンを検知:](/docs/apm/distributed-tracing/ui-data/understand-use-distributed-tracing-data#anomalous-spans) 一般的なインスタンスとの比較チャートを示した上で、一般的な行動と比較して遅いスパンは異常であるとマークされます。
* [クエリとダッシュボードをカスタマイズ:](/docs/insights/use-insights-ui/manage-dashboards/create-edit-copy-insights-dashboards) トレースデータのカスタムクエリを作成し、カスタムデータダッシュボードを作成することもできます。
* [アカウント全体でデータを表示:](/docs/apm/distributed-tracing/ui-data/additional-distributed-tracing-features-new-relic-one) すべてのアカウントおよびアプリケーションにまたがるトレースの全体的な見解を得られます。
* [プログラムに基づきトレースのクエリを実行:](/docs/apis/graphql-api/tutorials/query-distributed-trace-data-using-graphql-api) NerdGraph APIエクスプローラーでGraphQLを使用して、ディストリビューティッド（分散）トレーシングのデータのクエリを行います。
