<Callout variant="important">
  2021年03月22日の時点で、Insightsをアップグレードしてウェブとモバイルの体験を向上しています。InsightsのすべてのURLは、New Relic Oneの対応する[ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards)に自動的にリダイレクトされます。この移行について、およびこの移行を容易に計画することの詳細については、[Explorers Hubの投稿](https://discuss.newrelic.com/t/important-insights-upgrade-to-nr1-dashboards/127823)をご覧ください。
</Callout>

[New Relicのブラウザ監視](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser)は、一部のUIで表示され、[クエリとチャート作成](https://docs.newrelic.co.jp/docs/using-new-relic/data/understand-data/new-relic-data-types)にも使用できる[イベントデータ](https://docs.newrelic.co.jp/docs/using-new-relic/data/understand-data/query-new-relic-data)をレポートします。次の表からイベント名を選択して属性を確認します。

<table>
  <thead>
    <tr>
      <th style={{ width: "250px" }}>
        イベント
      </th>

      <th>
        説明
      </th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>
        [`PageView`](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        `PageView`は、各ブラウザページ読み込みについて地理およびタイミングのデータを追跡します。
      </td>
    </tr>

    <tr>
      <td>
        [`PageViewTiming`](https://docs.newrelic.co.jp/attribute-dictionary/pageview)
      </td>

      <td>
        `PageViewTiming`は個々のデータポイントを、それが利用できるようになったらすぐに別のイベントとして送信します。これは、同期的かつ動的なページにおいて役に立ちます。

        このイベントを使用するには、Browserエージェントバージョン1149以上を保有していなければなりません。詳しくは、[`PageViewTiming`クエリ例を参照してください](/docs/browser/new-relic-browser/page-load-timing-resources/pageviewtiming-async-or-dynamic-page-details)。
      </td>
    </tr>

    <tr>
      <td>
        [`PageAction`](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        `PageAction`イベントは、ページで実行された各アクションのデータを追跡します。その属性は、アプリケーションデータと地理データに加え、ユーザーのブラウザの寸法、セッションID、および参照URLとページURLに関するデータも提供します。
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserInteraction` (SPA)](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        `BrowserInteraction`には、いくつかのPageView属性、およびシングルページアプリケーション (SPA) に固有の属性が含まれています
      </td>
    </tr>

    <tr>
      <td>
        [`AjaxRequest` (SPA](https://docs.newrelic.co.jp/attribute-dictionary))
      </td>

      <td>
        `AjaxRequest`イベントは、`BrowserInteraction`イベント中にAjaxリクエストが発生するときに自動で作成されます。このイベント属性は、地理情報とブラウザ情報を追跡します。
      </td>
    </tr>

    <tr>
      <td>
        [`BrowserTiming` (SPA)](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        `BrowserTiming`は、カスタム[createTracer](https://docs.newrelic.co.jp/docs/browser/new-relic-browser/browser-agent-apis/new-relic-spa-api) SPA APIメソッドを使用して開始されたブラウザインタラクションに関するSPAタイミングデータをキャプチャするカスタムイベントです。`BrowserTiming`には、他のイベント (特に`AjaxRequest`) によって使用されるものと同じ属性の多くが含まれています。
      </td>
    </tr>

    <tr>
      <td>
        [`JavaScriptError`](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        JavaScriptエラーが発生すると、`JavaScriptError`イベントが作成されます。`JavaScriptError`は、JavaScriptエラーデータ、およびブラウザインタラクションデータをキャプチャします。詳細については、Browser UIの[JSエラーページ](https://docs.newrelic.co.jp/docs/browser/new-relic-browser/browser-pro-features/javascript-errors-page-examine-errors-over-time)と、BrowserおよびSPAのイベントのための[NRQLクエリの例](https://docs.newrelic.co.jp/docs/insights/nrql-new-relic-query-language/nrql-query-examples/insights-query-examples-new-relic-browser-single-page-app-data)を参照してください。
      </td>
    </tr>

    <tr>
      <td>
        [`Span`](https://docs.newrelic.co.jp/attribute-dictionary)
      </td>

      <td>
        [ディストリビューティッド（分散）トレーシングがNew Relic Browserについて有効](/docs/browser/new-relic-browser/browser-pro-features/browser-data-distributed-tracing)な場合、`スパン`データが報告されます。
      </td>
    </tr>
  </tbody>
</table>
