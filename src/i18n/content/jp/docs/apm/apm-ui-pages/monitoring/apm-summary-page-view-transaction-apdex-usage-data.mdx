APMの**Summary**ページには、選択されたアプリに関する一般的な情報が含まれています。これらの情報には、[ウェブトランザクション](/docs/apm/transactions/intro-transactions/transactions-new-relic-apm)と[ウェブ以外のトランザクション](/docs/apm/transactions/intro-transactions/monitor-background-processes-other-non-web-transactions)、[Apdex （Application Performance Index）](/docs/site/apdex-measuring-user-satisfaction)スコア、CPU使用率、スループット（1分あたりのリクエスト数（ **rpm** ））、トランザクションタイム、エラー率、アプリケーションのアクティビティ、[ホスト](/docs/accounts-partnerships/accounts/account-billing-usage/compute-unit-pricing-host-pricing#understand)が含まれます。

<Callout variant="tip">
  すべてのアプリケーションおよびサービスに関する大まかな概要を得るには、[エンティティエクスプローラー](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer)を利用してください。
</Callout>

## アプリケーションの概要ページを表示 [#overview]

**Summary**ページを表示に刷るには、次の2つの方法があります。

* エンティティエクスプローラー：**[one.newrelic.com](https://one.newrelic.com) >** Entity explorer > （アプリケーションを選択）の順に移動します。
* APM：**[one.newrelic.com](https://one.newrelic.com) > APM > （アプリケーションを選択）** の順に移動します。

詳しくは、New Relic Oneの[中核となるUIコンポーネントの操作](/docs/new-relic-one/use-new-relic-one/get-started/new-relic-one-core-ui-components)に関するドキュメンテーションを参照してください。

    ブラウザの詳細を表示するには、[ブラウザ設定](/docs/new-relic-browser/browser-settings)からこの機能を有効にする必要があります。ただし、ご使用のアプリケーションがブラウザのデータを報告したことが**一度もない**場合は、まずアプリケーションの設定から有効にする必要があります。

    1. **[one.newrelic.com/apm](https://one.newrelic.com) > （アプリケーションを選択） > Settings > Application**の順に移動します。
    2. **New Relic Browser**セクションから、**Enable browser monitoring?**チェックボックスをオンにします。
    3. ブラウザの監視とアプリケーションサーバーのリクエストについて、Apdex値を選択するか、デフォルトのままにします。
    4. オプション：[地理](/docs/browser/new-relic-browser/additional-standard-features/browser-geography-webpage-performance-location)ページで、強調表示するブラウザ監視対象の国または地域を5つまで選択します。
    5. **Save application settings**を選択します。
    6. 追加の機能を有効にするには、**Browser > （選択したアプリ） > Settings**から[標準手順](/docs/new-relic-browser/browser-settings)に従ってください。

    New Relic Browserのインストゥルメンテーションをセットアップした後、APM**概要**ページには、サマリー情報の他に、アプリの対応する[ブラウザ**概要**ページ](/docs/new-relic-browser/browser-overview)に関する詳細情報へのリンクが表示されます。

    * ブラウザの[ページロードタイム](/docs/new-relic-browser/page-load-timing-process)でチャートの詳細を表示するには、メインチャートの**ブラウザ**リンクを選択します。
    * ブラウザのApdexスコアを表示するには、Apdexチャートの**Browser**リンクを選択します。

## アプリケーションのパフォーマンスをリソースにリンクする [#infra-server]

APM**概要**ページには、[ホスト](/docs/accounts-partnerships/accounts/account-billing-usage/compute-unit-pricing-host-pricing#understand)でお使いのアプリケーションのインスタンスに関する次の平均が、テーブル形式で掲載されています。

* Apdex
* レスポンスタイム
* スループット
* エラー率
* CPU使用率
* メモリ

<Callout variant="tip">
  アプリケーションが1つのCPUコアで動作しているかのように、CPU使用率が計算されます。この計算の詳細については、[CPU使用率が100％を超える](/docs/apm/new-relic-apm/troubleshooting/cpu-usage-over-100#h2-apps)を参照してください。
</Callout>

  <Collapser
    id="host_details"
    title="Infrastructure内の詳細を調べる"
  >
    環境内におけるアプリケーションのパフォーマンスの状況を完全に理解するため、New Relic APMには、アプリケーション内部からだけでなく、[Infrastructure](/docs/infrastructure/new-relic-infrastructure/getting-started/welcome-new-relic-infrastructure)エージェントを使ってアプリケーション外部からのパフォーマンスを表示するオプションがあります。

    リソースの観点から詳細な情報を表示するには、任意のホストリンクをクリックします。このリンクをクリックすると、Infrastructureの[**Compute**ページ](/docs/infrastructure/new-relic-infrastructure/infrastructure-ui-pages/infrastructure-compute-page-measure-resource-usage)に直接アクセスできます。

    ![New Relic APM：Infrastructureのリンク](./images/crop-apm-overview-hosts112116.png "New Relic APMの概要：ホストのリンク")

    クリックしても、**compute**データがすぐに表示されないことがあります。この問題が発生した場合、プロンプトに従ってアカウントを検証し、Infrastructureエージェントの変換プロセスを完了します。さらに支援が必要な場合は、[**support.newrelic.com**](https://support.newrelic.com)でサポートを得てください。
  </Collapser>
