Syntax

add_user_attribute(key, value)

(Deprecated.) Adds a custom attribute to a transaction.

Description

This call is deprecated, and aliases to add_custom_parameter(). Use add_custom_parameter() instead.

This call records an attribute (a key/value pair attached to a transaction). The key must be a string. The value must be either a string, integer, float, or boolean.

Parameters

Parameter

Description

key

string

Required. The key name. Current allowed values for the key are:

  • user
  • account
  • product

value

string, integer, float, boolean

Required. The string value to add to the current transaction. Only the first 255 characters are retained.

Return values

None.
