Decisions about translation are important whenever you work on new feature documentation or make significant improvements to existing documentation. We encourage you to review the translation approval criteria in the Docs site's style guide, and to consider whether your work is a good candidate for translation. (We do not translate release notes, data dictionary values, hidden docs, or docs in private beta or limited release.)

The procedures in this doc explain several translation scenarios and are intended for tech writer use only. If you are not a tech writer and have questions about requesting a doc for translation, contact @hero in the #documentation Slack channel.

Translation project background [#project]

All of the New Relic documentation that exists on our English docs site (docs.newrelic.com) also exists on our Japanese docs site (docs.newrelic.co.jp). However, not all of the docs have been translated into Japanese on that site.

An initial set of key docs on the Docs site was translated into Japanese during 2018, and requests for new or existing docs have continued since then. In addition, sets of 200 core docs have been identified for translation projects in any language, such as French and German.

In Drupal Edit mode, every doc's Translation info section indicates its current status for translation.

For more information about translated docs (including toggling between the two sites, viewing the translation status, selecting translation criteria, and getting data about translated docs), see the Docs translation documentation in the style guide.

Clone an existing doc [#clone-translation]

Any time you clone an existing doc, always do the following. This applies whether or not you want to request your new (cloned) doc for translation:

  1. Always check whether the doc you are cloning has already been translated. Either search the Translation index in Drupal for any docs, or view the Translation info section in the doc you are cloning.
  2. If the doc that you are cloning has already been translated, be sure to clear the Translation info fields in the new doc before you save it.
  3. If you want to request the new (cloned) doc to be translated, continue with the procedure to request a new doc to be translated.

Request a new doc to be translated [#new-translation]

This procedure is for GA docs with full URLs on the Docs site. We do not translate release notes, data dictionary values, hidden docs, or docs in private beta or limited release.

If you are writing a new doc, or if you are the Tech Docs liaison for a new feature with multiple important new docs, follow this procedure:

  1. Use the translation criteria to evaluate whether there is a good business case to translate the doc. If yes, continue. If no, stop here.

  2. In Drupal: Edit the doc you want to have translated, and scroll to the doc's Translation info section:

    • Target languages: Japanese
    • Translation status: Wish listed

  3. In Drupal: Scroll to the bottom of the doc, make the following changes, then save as Published:

    • URL path settings: Make sure Generate automatic URL alias is checked.
    • Moderation notes: Note your Jira, and then use Keyboard Maestro's kkcriteria to identify the translation criteria you used with your PM or stakeholders.

  4. Liaison: Based on your conversation with your PM and stakeholders, add the docs to your Docs liaison spreadsheet's Translation Impact tab.

The translation team then uses the doc's For quote status to prioritize and get budget approval to translate. When translation is completed, the doc's status is updated to Completed.

Merge an existing translated doc into an untranslated doc [#merge-translated]

This procedure is for GA docs with full URLs on the Docs site. We do not translate release notes, data dictionary values, hidden docs, or docs in private beta or limited release.

If you merge an existing translated doc into a new or existing doc that has not been translated, follow this procedure:

  1. In your Jira comments: Before you delete and redirect the existing translated doc into the other doc, note the full URL of the translated doc and the full URL of the doc where it will be merged. Tag Julien Mira in your Jira.

  2. In Drupal: Edit the doc you now want to request for translation, and scroll to the doc's Translation info section:

    • Target languages: Japanese
    • Translation status: Wish listed

  3. In Drupal: Scroll to the bottom of the doc, make the following changes, then save as Published:

    • URL path settings: Make sure Generate automatic URL alias is checked.
    • Moderation notes: Note your Jira and the previously translated doc that was merged into this one.

  4. Liaison: Based on your conversation with your PM and stakeholders, add the docs to your Docs liaison spreadsheet's Translation Impact tab.

The translation team then uses the doc's For quote status to prioritize and get budget approval to translate. When translation is completed, the doc's status is updated to Completed.

Merge an untranslated doc into a translated doc [#merge-untranslated]

If you merge content from an untranslated doc into a doc that has already been translated, the translation service automatically handles the new content. No additional steps are necessary.

Exception: If you are the Tech Docs liaison for a new feature with multiple existing docs that have already been translated, and if those translated docs will need significant updates, follow this procedure:

  1. Add the translated docs to your Docs liaison spreadsheet's Translation Impact tab.

  2. In Drupal: Edit the doc you want to have re-translated to include the new content, and scroll to the docs Translation info section:

    • Target languages: Japanese
    • Translation status: Completed

  3. In Drupal: Scroll to the bottom of the doc, make the following changes, then save as Published:

    • URL path settings: Make sure Generate automatic URL alias is checked.
    • Moderation notes: Note your Jira and the updated doc's size identified in your Docs liaison spreadsheet.

  4. Liaison: Add the updated docs' URLs to your Docs liaison spreadsheet's Translation Impact tab.

The translation team then uses the doc's For quote status to prioritize and get budget approval to translate. When translation is completed, the doc's status is updated to Completed.

Delete a translated doc [#delete-translated]

Before you redirect and delete an existing doc:

  1. In your Jira comments: Note the full URL of the translated doc and the full URL of the doc where it has been redirected. Tag Julien Mira in your Jira.
  2. Liaison: Add the deleted doc's URL to your Docs liaison spreadsheet's Translation Impact tab.
