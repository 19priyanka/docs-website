The Docs site has two kinds of release notes:

  • Agent release notes (for downloadable, versioned New Relic software)
  • Platform release notes (for other SaaS-based software like UI changes)

Review the recommended best practices for New Relic release notes. Then, to add either type of release note to the site:

  1. Create a new release note:

    • To clone an existing release note as a template: From the release note you want to clone, select Clone content below the Page tools menu bar. Ensure you update the title, version numbers, dates, and download links.
    • To create a new note: From the Drupal menu bar, select Shortcuts > Add content > Add agent release notes or Add platform release notes. Use the placeholder text as a starting template for your note.

  2. Fill out the required fields for either release note type:

    Agent release notes

    Field

    Description

    Page contents

    Body of the release note containing the info for the release. Only use <h3> tags for headings.

    The Docs site has a known bug with release notes that affects the first line's formatting. To work around this, ensure your first line includes <h3> tags (### in Markdown); for example, ### Notes.

    Page location

    Specify one primary category for this content to live (shows up in sidebar categories, RSS feeds, and menu pages).

    Release version

    The version number of the release.

    Released on

    The time stamp of the release. We only display the date stamp on the front-end. We use the time stamp for sorting.

    Title

    Title of the release. Mainly used on the site to generate path alias, to display a meaningful title on the menu page and single node page.

    Platform release notes

    Field

    Description

    Page contents

    Body of the release note containing the info for the release. Only use <h3> tags for headings. Markdown is not supported.

    The Docs site has a known bug with release notes that affects the first line's formatting. To work around this, ensure your first line includes <h3> tags.

    Page location

    Primary category/categories for this content to live (shows up in sidebar categories, RSS feeds, and menu pages). This content type makes use of multiple values for this taxonomy.

    Product

    Affected product(s). Multiple values possible.

    Used as a filter on the filterable page.

    Release name

    Name of the release. Generally, the syntax is: [YYYY-MM-DD] [Name of product]: [Headline item]. This value is used when generating the path alias (similar to Title field in other content types).

    Release impact

    The type of impact this release may have on end-users. Multiple values possible.

    Used as a filter on the filterable page.

    We designed platform release notes to support multiple types of impact per release note so PMs can batch all releases into one broader note.

    Released on

    The time stamp of the release. We only display the date stamp on the front-end. We use the time stamp for sorting.

  3. To save as a draft, select Save.

    OR

    To publish your release note, change the Moderation state to Published, and then select Save.
