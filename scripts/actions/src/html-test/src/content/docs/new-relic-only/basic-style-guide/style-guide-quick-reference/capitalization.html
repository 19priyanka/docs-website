This explains how to decide what to capitalize in a document's title, headings, products, features, and other elements of the page.

Use sentence case in headings [#heading_caps]

Use sentence case for headings, including category headings and document titles. With sentence case, capitalize only the first letter of:

  • The first word
  • Proper nouns
  • Acronyms and abbreviations

Exceptions:

  • If the heading is a code term, such as a variable or function, then capitalize it exactly as it is used in the code; for example: noticeError.
  • If the heading includes a colon, follow the Microsoft Style Guide for titles and headings, and capitalize the first word that appears after the colon; for example: APM Error profiles: Troubleshoot trends.

Products and features [#features]

Item

Example

Use title case for products.

New Relic Infrastructure

Do not capitalize features.

"Use transaction traces to..." not "Use Transaction Traces to..."

UI elements and UI page paths [#ui]

Item

Example

Use sentence case and bold for UI elements, even if the UI element is in a different case in the UI.

"From the Transactions page, select Transaction traces and..."

Use sentence case and bold for each element in a path that references UI pages.

Go to rpm.newrelic.com/apm > Transactions > Transaction traces > (select a trace) > Another thing.

Watermarks

Item

Example

Use all caps for BETA or NR ONLY.

<div id="watermark">NR ONLY</div>

Otherwise use sentence case.

<div id="watermark">Legacy</div>

Include break (br /) for longer watermarks.

<div id="watermark">Limited <br /> release</div>
